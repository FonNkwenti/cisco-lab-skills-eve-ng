# cisco-lab-skills

AI agent skills for Cisco certification lab generation. Used as a **git submodule** at `.agent/skills/` inside every certification lab project.

One repo, shared across all exam series (ENARSI, SPCOR, ENCOR, etc.). Fix a skill once — pull the update into every exam repo.

## Skills

| Skill | What It Does |
|-------|-------------|
| `chapter-topics-creator` | Plans a chapter — generates `baseline.yaml` + backfills `chapter-spec.md` |
| `chapter-builder` | Generates all labs in a chapter with config chaining |
| `lab-workbook-creator` | Generates a single lab (workbook, configs, topology, scripts) |
| `fault-injector` | Creates automated Netmiko fault injection scripts |
| `mega-capstone-creator` | Generates the multi-domain final capstone spanning all chapters |
| `drawio` | Visual style guide + `generate_topo.py` for topology diagrams |
| `eve-ng` | EVE-NG constraints and hardware selection reference (QEMU/IOL/Dynamips on Intel) |
| `gns3` | **DEPRECATED** — Legacy GNS3/Apple Silicon archive (c7200/c3725, read-only) |
| `cisco-troubleshooting-1` | Structured 4-phase network troubleshooting methodology |

## Building Labs for a CCNP Exam (ENCOR Example)

This section walks through the full workflow for building a chapter of labs inside a
certification exam repo. ENCOR is used as the example — the same steps apply to any exam.

### 1. Create an exam repo and mount this skill library

```bash
mkdir ccnp-encor-labs && cd ccnp-encor-labs
git init
git submodule add https://github.com/FonNkwenti/cisco-lab-skills-eve-ng.git .agent/skills
```

Your exam repo structure will look like:

```
ccnp-encor-labs/
├── .agent/
│   └── skills/           ← this repo, mounted as submodule
├── specs/
│   └── ch01-ospf/
│       └── chapter-spec.md    ← you write this (blueprint bullets)
├── labs/
│   └── ch01-ospf/
│       ├── baseline.yaml      ← generated by chapter-topics-creator
│       ├── lab01/
│       │   ├── workbook.md
│       │   ├── initial-configs/
│       │   ├── solutions/
│       │   ├── topology.drawio
│       │   ├── setup_lab.py
│       │   └── scripts/fault-injection/
│       └── lab02/ ...
└── CLAUDE.md                  ← imports memory/CLAUDE.md from skills submodule
```

### 2. Get the current exam blueprint and write chapter-spec.md

> **Important:** Cisco updates exam blueprints. Always pull the current version from the
> official exam page — do not rely on old PDFs, memory, or third-party study guides.
> The CCNP ENCOR blueprint (350-401) was revised in 2025; earlier versions will produce
> labs for retired or reweighted objectives.

1. Go to [Cisco's exam page](https://www.cisco.com/c/en/us/training-events/training-certifications/exams.html), search for your exam, and download the current exam topics PDF.
2. Copy the relevant chapter bullets into `specs/ch01-ospf/chapter-spec.md`:

```markdown
# Chapter 01 — OSPF

## Source
Exam: CCNP ENCOR 350-401
Blueprint version: [paste version/date from Cisco PDF]
Retrieved: [YYYY-MM-DD]

## Exam Objectives
- 3.x Implement OSPF for IPv4 and IPv6
- 3.x Describe OSPF neighbor establishment and LSA types
- 3.x Configure and verify OSPF areas (backbone, stub, NSSA)
- 3.x Tune OSPF (timers, cost, authentication)
```

When you ask the agent to plan a chapter, paste the blueprint bullets directly into the prompt
if `chapter-spec.md` does not yet exist. The `chapter-topics-creator` skill will stop and ask
if it cannot confirm the blueprint is current.

### 3. Plan the chapter — run chapter-topics-creator

Ask the agent: *"Plan chapter 01 OSPF for CCNP ENCOR"*

The `chapter-topics-creator` skill reads `chapter-spec.md` and generates:
- A lab series plan (e.g. 7 labs: Foundation → Intermediate → Advanced → Capstone I + II)
- `labs/ch01-ospf/baseline.yaml` — shared topology, IP plan, device list for all labs

### 4. Generate labs — run lab-workbook-creator or chapter-builder

**One lab at a time (recommended):**
Ask: *"Generate lab 01 for chapter 01 OSPF"*

The `lab-workbook-creator` skill produces:
- `workbook.md` — student workbook with 9+ sections, theory, tasks, troubleshooting
- `initial-configs/*.cfg` — startup configs for each device
- `solutions/*.cfg` — reference solution configs
- `topology.drawio` — Cisco-icon topology diagram
- `setup_lab.py` — Netmiko script to push configs to EVE-NG
- `scripts/fault-injection/` — automated fault injection + solution scripts

**Entire chapter at once:**
Ask: *"Generate all labs for chapter 01 OSPF"*
The `chapter-builder` skill chains all labs in sequence (each lab's solutions become the next lab's initial-configs).

### 5. Create the lab in EVE-NG

1. Open EVE-NG web UI → create a new lab (e.g., `encor-ch01-ospf-lab01.unl`)
2. Add nodes matching `baseline.yaml` (platform: `iosv` for routing, `iosvl2` for switching)
3. Connect links as shown in `topology.drawio`
4. Start all nodes and wait for boot

### 6. Discover console ports and push configs

Console ports are assigned dynamically by EVE-NG. Find them via:
- **Web UI:** right-click each node → Properties → note the telnet port
- **REST API:** `GET http://<eve-ng-ip>/api/labs/<lab-path>/nodes`

Then push initial configs:

```bash
cd labs/ch01-ospf/lab01
pip install -r ../../../../.agent/skills/scaffolding/requirements.txt
python setup_lab.py --host 192.168.1.x
```

`setup_lab.py` accepts a `--host` argument for your EVE-NG server IP. Update `DEVICES` dict
with the ports discovered in step 5.

### 7. Do the lab / practice troubleshooting

- Open `workbook.md` and work through the tasks
- Use `scripts/fault-injection/inject_scenario_01.py` to break the network for troubleshooting practice
- Use `scripts/fault-injection/apply_solution.py` to restore it

### Summary

```
chapter-spec.md (you write)
    │
    ▼
chapter-topics-creator  →  baseline.yaml
    │
    ▼
lab-workbook-creator    →  workbook.md + configs + setup_lab.py
    │
    ▼
EVE-NG lab              →  create nodes, start, discover ports
    │
    ▼
setup_lab.py --host     →  push initial-configs to all devices
    │
    ▼
workbook.md             →  study + practice
```

## Adding to a new exam repo

```bash
cd /path/to/your-exam-repo
git submodule add https://github.com/FonNkwenti/cisco-lab-skills-eve-ng.git .agent/skills
git commit -m "chore: add cisco-lab-skills submodule"
```

## Cloning an exam repo (pulls skills automatically)

```bash
git clone --recurse-submodules /path/to/exam-repo
# OR if already cloned without --recurse-submodules:
git submodule update --init
```

## Updating a skill

### Step 1 — Edit the skill in this repo

```bash
cd C:\Users\Nkwenti\Documents\Labs\Cisco\cisco-lab-skills-eve-ng
# edit chapter-topics-creator/SKILL.md, lab-workbook-creator/SKILL.md, etc.
git add .
git commit -m "fix(lab-workbook-creator): capstone clean-slate initial-config logic"
```

### Step 2 — Pull the update into each exam repo

```bash
# ENARSI (or whichever exam repo needs the update)
cd C:\path\to\exam-repo
git submodule update --remote .agent/skills
git add .agent/skills
git commit -m "chore: update skills submodule"
```

Each exam repo pins skills to a **specific commit**. `git submodule update --remote` advances the pin to the latest. This means:
- You can update ENARSI skills without touching SPCOR until you're ready
- If an update breaks something, the other exam repos are unaffected
- `git log .agent/skills` in any exam repo shows the full skills change history

## Checking which commit each repo uses

```bash
# Inside any exam repo
git submodule status
# Output: abc1234 .agent/skills (v1.2.0)
#          ↑ commit hash pinned by this exam repo
```
